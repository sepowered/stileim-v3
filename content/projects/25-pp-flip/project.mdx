## **이번 프로젝트는 이런 것이었어요**


- Flip은 숏폼으로 운동 정보를 소비하지만 실제 운동으로 이어지지 않는 병목을 해결하기 위해 기획한 **취향 기반 숏폼 건강관리 플랫폼**이에요.
- 운동을 시작하려는 사용자들은 TikTok/Instagram Shorts에서 정보를 많이 얻지만, 콘텐츠를 저장해도 ‘다시 실행하기 어렵고’, ‘나에게 맞는 난이도를 판단하기 힘들며’, 결국 실행 가능성이 낮다는 문제를 겪고 있었어요.
- 이 프로젝트에서는 **숏폼 콘텐츠 → 운동 동작 추출 → 루틴 자동 생성 → 난이도 조정 → 추천**이라는 흐름을 통해, 기존 숏폼 플랫폼이 해결하지 못한 **실행 가능성** 문제를 풀고자 했어요.

## **이 과정에서 이런 역량을 활용했어요**


- **문제 구조화 역량**
    
    숏폼 소비–실행의 병목을 행동 단위로 정의하고 구조적 문제로 전환했어요.
    
- **사용자 리서치 역량**
    
    Town Watching(탄천·헬스장) + 7명의 인터뷰를 통해
    
    데스크 리서치 가설을 검증하고 인사이트를 도출했어요.
    
- **플로우 설계 & 알고리즘 사고**
    
    사용자 행동 데이터를 기반으로
    
    “추천 강도 조절 알고리즘”을 직접 설계하는 첫 경험을 했어요.
    
- **서비스 기획 & IA 설계**
    
    콘텐츠 소비·루틴 생성·코칭·데이터 입력 등 전체 플로우를
    
    IA/Flow/Wireframe으로 구성했어요.
    
- **도메인 리서치**
    
    운동 난이도, 운동 동작 유형, 건강관리의 사용자 여정을
    
    생소한 도메인에서도 빠르게 학습하고 구조화했어요.
    

## **프로젝트에서 이런 역할을 맡았어요**


- **문제정의 및 핵심 가설 설정**
- **사용자 리서치 설계·수행 (인터뷰 및 타운워칭 주도)**
- **경쟁사 분석(틱톡/인스타/밋업) 및 시장 구조 분석**
- **서비스 IA·플로우·알고리즘 설계**
- **핵심 와이어프레임 제작**
- **‘운동 루틴 생성 로직’ 및 ‘난이도 조절 알고리즘’ 설계**

## **프로젝트를 진행하며 이런 문제가 보였어요**


리서치 과정에서 세 가지 큰 병목이 반복적으로 나타났어요.

### **1) 숏폼 기반 운동 정보는 많지만 “실행으로 연결되지 않는다”**

- 저장은 하지만 실제로 다시 보지 않음
- 루틴으로 연결되지 않아 ‘그때 본 운동’이 그대로 증발함

### **2) 초보자는 ‘난이도 판단’이 어려워 시작 장벽이 높다**

- 어떤 운동이 쉬운지 어려운지 감이 없음
- 기존 운동 루틴이 깨질까 불안해 새로운 동작을 시도하지 않음

### **3) 운동 지속성의 핵심은 ‘성공 경험’인데, 대부분 실패한다**

- 너무 어려운 루틴을 선택해 금방 포기
- 단계 조절 기능 부재

## **그래서 이런 방식으로 문제를 바라봤어요**


이 프로젝트는 단순히 운동 콘텐츠를 보여주는 앱이 아니라, ‘행동 변화를 만드는 엔진’을 설계해야 했어요.

### **문제 구조화 (Behavior → Bottleneck → Mechanism)**

1. **Behavior**
    - 사용자는 숏폼에서 운동 정보를 얻는다 → 저장한다 → 다시 실행하지 않는다.
2. **Bottleneck**
    - 저장된 운동이 동작 단위로 분해되지 않음
    - 나에게 맞는 난이도인지 판단할 근거 없음
    - 저강도 실패 → 동기 저하 → 루틴 붕괴
3. **Mechanism (기저 메커니즘)**
    - 실행률을 높이기 위해서는 **콘텐츠 단위(Shorts)를 운동 단위(Exercise Atom)로 변환하고,**
    - **사용자 신체 데이터에 맞춰 조정하는 “행동 조절 메커니즘”이 필요하다.**

## **이 문제를 이렇게 해결해봤어요**


이 프로젝트의 핵심은 ‘데이터 기반 루틴 생성 알고리즘’을 설계한 것이었어요. 이는 제가 처음으로 기획을 시도한 알고리즘이며, 기획자로서 중요한 성장이었어요.

1. **숏폼 기반 운동 동작 추출**

콘텐츠를 저장하면 다음이 자동 수행돼요:

- 해시태그/텍스트 기반 운동 유형 파악
- 동작 단위 분해(예: 스쿼트·런지·힙힌지 등)
- 운동 목적·난이도 태깅

→ 숏폼 콘텐츠를 **‘실행 가능한 운동 데이터’로 변환**하는 과정이에요.

1. **사용자의 건강 데이터 기반 난이도 조절**
    
    앱 내에서 다음 데이터를 지속적으로 기록해요:
    
    - 체중 변화
    - 루틴 완수율
    - 저장한 운동 난이도 패턴
    - 운동 빈도
    
    이 데이터를 기반으로 루틴 강도를 조정해요.
    
    - 체중 증가 → 강도 하향
    - 체중 유지/감량 → 강도 유지·상향
    
    이 로직은 포트폴리오용 Flow에도 포함되어 있어요.
    
2. **자동 루틴 생성 — Routine Auto-Assembling**

Atom 단위 운동을 묶어서 오늘의 루틴을 자동 구성해요.

- 짧은 시간·초보자·무소음 등 필터 반영
- 저장 콘텐츠 기반으로 개인화
- 실패 시 한 단계 낮은 루틴 제공
    
    → “성공 경험”을 인위적으로 설계하는 구조
    
1. **추천 엔진 — Personalized Feed**
- 추천 콘텐츠는 다음 순서로 생성돼요.
    - 저장 콘텐츠의 해시태그
    - 사용자 체중 변화
    - 난이도 스코어
    - 사용자가 자주 보는 운동 유형

→ 행동 변화(Behavior Change)를 유도하는 콘텐츠 노출

## **이 경험을 통해 이런 것들을 배웠어요**


### **사용자 행동을 구조화하고, 이를 기반으로 의사결정 로직을 설계하는 방법을 배웠어요.**

- 숏폼 소비 → 저장 → 실행이라는 행동 흐름을 단계별로 분석하고, 사용자의 상태(체중 변화·완수율 등)에 따라 루틴 강도를 조절하는 규칙 기반 구조를 설계(기획)하는 경험을 했어요.
- 비록 엔지니어링 관점의 데이터 모델링을 직접 수행한 것은 아니지만, **사용자 행동 데이터를 어떤 기준으로 수집·해석해야 하는지**를 사고하는 기획자로서의 로직 설계(기획) 경험이었어요.

### **기획자로서 첫 알고리즘 기획 경험은 큰 전환점이 되었어요.**

- 데이터 → 조건 → 추천
- 신체 데이터 기반 의사결정
- 난이도 조절 로직

이를 직접 설계해보면서 “제품이 돌아가는 구조를 만드는 사람”의 역할을 체험했어요.

### **도메인 지식이 없더라도 ‘맵핑–가설–검증’ 프로세스를 따르면 충분히 유효한 인사이트를 만들 수 있다는 자신감을 얻었어요.**

### **결국, 사용자에게 필요한 것은 ‘기능’이 아니라 ‘실행 가능성’이라는 걸 다시 배웠어요.**
